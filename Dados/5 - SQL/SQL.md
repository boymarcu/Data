## O que é SQL?
- Significa Structured Query Language. 

É uma maneira de interagir com bancos de dados e tabelas relacionais de uma forma que permite que nós coletemos informações específicas e significativas.

### O que é um banco de dados relacional?
- Não há uma definição única para a palavra _banco de dados_.

Bancos de dados são sistemas de gerenciamento de informações.
Os bancos de dados podem ter quantidades variadas de estrutura impostas aos dados.

Os bancos de dados relacionais são provavelmente o tipo mais conhecido de banco de dados. Em sua essência, os bancos de dados relacionais são compostos de _tabelas_, que são muito parecidas com planilhas. Cada coluna da tabela tem um nome e um tipo de dados (texto, número, etc.), e cada linha na tabela é uma instância específica de tudo o que a tabela é "sobre".
A parte "relacional" vem com regras específicas sobre como conectar dados entre diferentes tabelas.

Imagine um banco de dados como uma coleção de planilhas do Excel.

- **Tabela (Table):** É como uma planilha individual. Por exemplo, você pode ter uma tabela `Usuarios` e outra tabela `Produtos`.
    
- **Coluna (Column):** É um cabeçalho na sua planilha. Na tabela `Usuarios`, você teria colunas como `ID`, `Nome`, `Email`.
    
- **Linha (Row):** É um registro único de dados. É uma linha na sua planilha. Por exemplo, uma linha na tabela `Usuarios` seria: `1`, `Ana Silva`, `ana@email.com`.

### O que é um ERD?
- Significa Diagrama de Relacionamento de Entidade

É uma representação visual das relações entre todas as tabelas relevantes em um banco de dados.
